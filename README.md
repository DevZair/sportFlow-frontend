# SportFlow Frontend

Это клиентская часть (Frontend) системы управления спортивными тренировками (SportFlow), разработанной в рамках курсового проекта для **ТОО «ASTANA IT UNIVERSITY»**.

## 📌 Описание
SportFlow — это современное веб-приложение для спортивных секций, предоставляющее интерфейсы как для учеников (бронирование тренировок, история посещений), так и для тренеров (создание секций, планирование занятий, учет посещаемости).

Интерфейс спроектирован с учетом современных трендов дизайна (glassmorphism) и поддерживает многоязычность (русский и казахский).

## 🚀 Основной функционал
- **Многоязычность**: Полный перевод всего интерфейса (RU/KZ) через `LangContext`.
- **Ролевая система**: Динамическое изменение интерфейса в зависимости от роли (Ученик / Тренер / Администратор).
- **Студент**: Просмотр расписания, бронирование мест, прогресс-бар посещаемости в личном кабинете.
- **Тренер**: Создание профилей секций, планирование тренировок по дням недели, учет посещаемости каждого студента на конкретную дату.
- **Современный UI/UX**: Адаптивный дизайн, карточки с hover-эффектами, лоадеры во время загрузки (предотвращение двойных кликов).

## 🛠 Технологии
- **React.js** (создан через Vite)
- **React Router DOM** (маршрутизация)
- **Axios** (работа с REST API, перехватчики запросов для JWT)
- **Lucide React** (набор SVG иконок)
- **CSS3 / Vanilla CSS** (Glassmorphism, кастомный дизайн без тяжелых библиотек UI)

## ⚙️ Установка и локальный запуск

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Создайте файл `.env` в корневой папке `frontend/` и укажите URL вашего backend сервера:
   ```env
   VITE_API_URL=http://localhost:3001/api
   ```

3. Запустите проект в режиме разработки:
   ```bash
   npm run dev
   ```

4. Откройте в браузере `http://localhost:5173` (по умолчанию для Vite).

## 🗂 Структура проекта
- `src/components` — переиспользуемые UI компоненты (кнопки, навигация, карточки, лоадеры).
- `src/context` — контексты React (Авторизация `AuthContext`, Языковой контекст `LangContext`).
- `src/i18n` — файлы локализации (словари для RU и KZ).
- `src/pages` — основные страницы приложения (Главная, Расписание, Панель тренера, Посещаемость и т.д.).
- `src/services` — настройка Axios для связи с сервером.

## 빌 Сборка для Production
```bash
npm run build
```
Собранные статические файлы будут находиться в папке `dist/`, готовые для разворачивания на Vercel, Netlify или любом другом хостинге статических сайтов.
